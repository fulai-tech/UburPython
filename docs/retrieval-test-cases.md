# UburNode 检索测试用例

本文档描述 `POST /api/audio/search` 检索流水线的测试用例，供手工验收（Apifox / curl）、日志核对与自动化回归对齐使用。

**相关文件**

| 文件 | 说明 |
|------|------|
| `docs/UburNode-HTTP-API.md` | HTTP 接口说明 |
| `docs/uburnode-openapi.yaml` | Apifox 导入 |
| `scripts/seed_es_test_data.py` | ES 测试数据种子 + 内置 `SEARCH_CASES` |
| `app/services/retrieval.py` | 四步检索流水线实现 |

---

## 1. 范围与约定

### 1.1 流水线顺序（不可调换）

1. **睡眠阶段过滤** — ES term 精确匹配；无命中 → 短路返回 `[]`
2. **内容形态准入** — 精确交集 或 向量余弦 ≥ `SIM_THRESHOLD`；无命中 → 短路返回 `[]`
3. **厌恶剔除 + 粗排** — `disliked_tags` 与内容标签有交集则剔除；无厌恶标签时原样透传
4. **精排截断** — 按 `match_count` 降序，取 `top_k`

### 1.2 环境与参数

| 项 | 约定 |
|----|------|
| Base URL | `http://localhost:8080` |
| 睡眠阶段枚举 | `放松` / `入睡` / `守护` / `清醒`（**非**「深睡」） |
| 模糊阈值 | `SIM_THRESHOLD=0.8`（`.env` / `app/core/config.py`） |
| 向量模型 | `BAAI/bge-small-zh-v1.5`，512 维 |
| 测试数据 | 执行 `scripts/seed_es_test_data.py` 灌入；前 48 条为固定锚点（`【测试】` 前缀） |

### 1.3 前置准备

```bash
# 项目根目录，已激活 .venv
.venv/bin/python scripts/seed_es_test_data.py --count 300 --verify
```

`--verify` 会跑 `scripts/seed_es_test_data.py` 内已实现的 `SEARCH_CASES` 回归。

### 1.4 用例字段说明

| 字段 | 含义 |
|------|------|
| **用例ID** | 唯一编号，格式 `RET-{步骤}-{序号}` 或 `RET-F-{序号}`（模糊专项） |
| **请求体** | 完整 JSON，可直接用于 Apifox / curl |
| **预期日志** | `logs/uburnode.log` 中四步计数值；用于排查「步骤3 剩余数 = 步骤2 通过数」等现象 |
| **预期响应** | HTTP 200，`code=200`，`data.results` 内容断言 |
| **自动化** | `✅` 已在 `SEARCH_CASES`；`⬜` 待补；`📋` 仅手工 |

### 1.5 日志关键字

```
检索步骤1/4 睡眠阶段过滤：候选数=N
检索步骤2/4 内容形态准入：通过数=N
检索步骤3/4 厌恶剔除+粗排：剩余数=N
检索步骤4/4 精排截断：top_k=K，输出数=N
```

## 4. 步骤3：厌恶剔除

步骤3 **只做剔除，不做截断**；`disliked_tags` 为空时直接透传，因此常见现象是「步骤3 剩余数 = 步骤2 通过数」。

剔除规则：`candidate.tags.content_labels()` 与 `disliked_tags` 有交集 → 剔除该条。

### RET-3-01 无厌恶标签 → 透传

| 项 | 内容 |
|----|------|
| 场景 | `disliked_tags=[]` 时步骤3 不删减 |
| 自动化 | 📋（多数历史回归请求均为空厌恶标签） |

**请求体**

```json
{
  "sleep_stage_tags": ["放松"],
  "content_tags": ["雨声", "森林", "白噪音"],
  "disliked_tags": [],
  "top_k": 50
}
```

**预期日志**

- 步骤2 通过数 = N
- 步骤3 剩余数 = **N**（与步骤2 相同）

**预期响应**

- 命中数 = N（若 N ≤ top_k）

---

### RET-3-02 厌恶标签命中 → 剔除

| 项 | 内容 |
|----|------|
| 场景 | 厌恶标签与文档内容标签有交集，对应音频被剔除 |
| 自动化 | 📋（2026-06-05 手工验证通过） |

**请求体**

```json
{
  "sleep_stage_tags": ["放松"],
  "content_tags": ["雨声", "森林", "白噪音"],
  "disliked_tags": ["森林"],
  "top_k": 50
}
```

**预期日志**

- 步骤2 通过数 = N
- 步骤3 剩余数 = **N - 1**（剔除含「森林」的 1 条）

**预期响应**

- 返回结果中 **不含** 内容标签含「森林」的音频
- 含「雨声」「白噪音」且不含「森林」的仍保留

---

### RET-3-03 厌恶标签无交集 → 不剔除

| 项 | 内容 |
|----|------|
| 场景 | 厌恶标签在候选内容标签中不存在 |

**请求体**

  "disliked_tags": ["不存在xyz"],

- 步骤3 剩余数 = 步骤2 通过数（例：4 → 4）

## 6. 模糊匹配专项

向量模糊仅在**精确未命中**时触发；阈值 `SIM_THRESHOLD=0.8`。

| 请求标签 | 文档标签 | 余弦相似度 | 预期 |

---

### RET-F-03 阿尔法波 ↔ α波（当前不命中，已知限制）

| 请求标签 | 文档标签 | 余弦相似度 | 预期 |
|----------|----------|-----------|------|
| 阿尔法波 | α波 | **0.644** | ❌ 低于 0.8，不命中 |

**说明**

- 希腊字母 `α` 与中文音译「阿尔法」在 `bge-small-zh` 中语义距离较远。
- 规范要求「同义词优先归一，不单独上向量」，但**同义词归一表尚未实现**。
- **不建议**为单一 case 降低 `SIM_THRESHOLD`（会误召回，如「海浪 ↔ 大海的波浪声」仅 0.703）。
- 后续方案：维护别名表，如 `{"α波","阿尔法","阿尔法脑波"} → "阿尔法波"`。

## 9. 与代码自动化映射

| 用例ID | `SEARCH_CASES.name`（seed 脚本） | 自动化 |
|--------|----------------------------------|--------|
| RET-1-01 / RET-2-01 | 放松+雨声精确 | ✅ |
| RET-2-02 | 入睡+雨声向量模糊 | ✅ |
| RET-2-03 | 放松+森林白噪音双标签 | ✅ |
| RET-1-03 | 入睡阶段查海浪 | ✅ |
| RET-5-01 | 守护仅睡眠阶段 | ✅ |
